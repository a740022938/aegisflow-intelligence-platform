## A. Gate Drilldown

**Source**: FactoryStatus 左侧 "阻塞 Gate" 卡片  
**Trigger**: 点击某行 blocked gate

**API**: `GET /api/gates?gate_name=<name>&status=blocked`

**Returns**:
```typescript
{
  ok: true,
  gates: [
    {
      id, gate_name, stage_name, entity_id, entity_type,
      status, fail_reasons_json, blocking_status, checked_at,
      check_results: [{ check, passed, reason }]
    }
  ],
  total: number,
  gate_name: string,
  status: string
}
```

**Panel Shows**:
- gate_name + stage_name
- total count
- list of blocked records with:
  - entity_id (truncated)
  - fail_reasons
  - checked_at
  - check_results
- "查看审计" button → links to Audit page

## B. Failure Trace

**Source 1**: FactoryStatus 中部 "最近失败任务" 卡片  
**Trigger**: 点击某行失败任务

**API**: `GET /api/factory/failures?time_range=24h&limit=20`

**Returns**:
```typescript
{
  ok: true,
  failures: [
    { id, name, status, error_message, updated_at,
      step_key, template_id, experiment_id }
  ],
  total: number,
  time_range: string
}
```

**Panel Shows**:
- job name + id
- status badge
- error_message (full)
- step_key
- updated_at
- experiment_id
- "查看审计" button → links to Audit filtered by job_id

---

**Source 2**: "失败原因聚合" 卡片  
**Trigger**: 点击某行 step_key

**API**: `GET /api/factory/failures?step_key=<key>&time_range=24h`

**Returns**: Same as above but filtered by step_key

**Panel Shows**:
- step_key
- count
- list of failed jobs with same detail as above

## G. Filter Implementation

### Backend: time_range SQL
```sql
-- 24h
datetime('now', '-24 hours')
-- 7d
datetime('now', '-7 days')
-- 30d
datetime('now', '-30 days')
```

### Frontend: filter state
```typescript
const [timeRange, setTimeRange] = useState<'24h'|'7d'|'30d'>('24h');
// passed as ?time_range=24h to API
```
