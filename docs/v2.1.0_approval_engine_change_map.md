## 1. 🔴 MUST — 必须改的文件

### 1.1 新建：`repo\apps\local-api\src\approvals\index.ts`
**原因**：当前无独立审批模块，审批逻辑散落在 workflow 模块内。
**内容**：
- Approval CRUD（基于 `approvals` 表）
- `createApproval()` — 创建审批请求
- `approveApproval()` / `rejectApproval()` — 审批动作
- `listApprovals()` / `getApprovalById()` — 查询
- `getPendingApprovals()` — 待审批列表

### 1.2 改：`repo\apps\local-api\src\workflow\index.ts`
**原因**：当前审批逻辑嵌入在 step 执行循环中，需要解耦。
**改动点**：
- `registerWorkflowRoutes()` 中的 `approve` 路由：调用新的 approvals 模块
- `runWorkflowJob()` 中的审批检查：改为查询 approvals 表而非只读 input_json
- 新增：审批请求创建逻辑（当 step 需要 approval 时自动创建 approval 记录）

### 1.3 改：`repo\apps\local-api\src\index.ts`
**原因**：所有 API 路由的唯一注册入口。
**改动点**：
- `import * as approvals` 新模块
- 注册审批 API 路由
- 初始化 `audit.setDatabase()`（已有，确认位置正确）

### 1.4 改：`repo\apps\local-api\src\audit\index.ts`
**原因**：审批动作需要写入审计日志。
**改动点**：
- 新增 `logApprovalCreated()` / `logApprovalAction()` 辅助方法
- 或直接在 approvals 模块中调用 `logAudit()`

### 1.5 改：`repo\packages\db\schema.sql`
**原因**：approvals 表需要升级字段。
**改动点**：
- 更新 `approvals` 表 DDL（增加 resource_type, workflow_job_id, job_step_id, reviewed_by, reviewed_at 等）
- ⚠️ 注意：实际建表逻辑在 `builtin-sqlite.ts`，schema.sql 只是原始参考

### 1.6 改：`repo\apps\local-api\src\db\builtin-sqlite.ts`
**原因**：运行时表创建/迁移的实际入口。
**改动点**：
- 更新 `approvals` 表的 ALTER TABLE 语句（新增列）
- 或新增 CREATE TABLE IF NOT EXISTS + 迁移逻辑

### 1.7 新建：`repo\apps\web-ui\src\pages\WorkflowJobs.tsx` 或新建 `Approvals.tsx`
**原因**：前端需要展示审批列表和审批操作界面。
**改动点**：
- 注册 `/workflow-jobs` 路由到 `App.tsx`（当前缺失）
- 在 WorkflowJobs 页面中，审批按钮改为调用新的 approvals API
- 或新建独立的审批中心页面

### 1.8 改：`repo\apps\web-ui\src\App.tsx`
**原因**：注册 WorkflowJobs / Approvals 路由。
**改动点**：
- 添加 `<Route path="workflow-jobs" element={<WorkflowJobs />} />`
- 如果新建审批页面，添加对应路由

### 1.9 改：`repo\apps\web-ui\src\services\api.ts`
**原因**：API service 层需要增加审批相关接口。
**改动点**：
- 新增 Approval 相关类型定义
- 新增 `approveStep()` / `rejectStep()` / `listApprovals()` 方法

## 6. 附：现有 approvals 表当前结构 vs 建议结构

### 当前（schema.sql）
```sql
CREATE TABLE IF NOT EXISTS approvals (
  id TEXT PRIMARY KEY,
  task_id TEXT NOT NULL,
  action TEXT NOT NULL,
  approver TEXT,
  status TEXT NOT NULL,
  comment TEXT,
  created_at TEXT NOT NULL
);
```

### 建议 v2.1.0 升级为
```sql
CREATE TABLE IF NOT EXISTS approvals (
  id TEXT PRIMARY KEY,
  -- 关联资源（支持 workflow job 和未来其他资源）
  resource_type TEXT NOT NULL DEFAULT 'workflow_job',  -- workflow_job / task / run / deployment
  resource_id TEXT NOT NULL,                            -- workflow_job.id
  step_id TEXT,                                         -- job_steps.id（可选，标识具体步骤）
  step_name TEXT,                                       -- 步骤名称（冗余，方便查询）
  -- 审批内容
  action TEXT NOT NULL,                                 -- approve / reject / request
  title TEXT,                                           -- 审批标题/描述
  -- 审批状态
  status TEXT NOT NULL DEFAULT 'pending',               -- pending / approved / rejected / expired / cancelled
  -- 审批人
  requested_by TEXT DEFAULT 'system',
  reviewed_by TEXT,
  reviewed_at TEXT,
  comment TEXT,
  -- 审批策略（预留）
  policy_type TEXT DEFAULT 'manual',                    -- manual / auto_approve / auto_reject / timeout
  timeout_seconds INTEGER DEFAULT 0,                    -- 0 = 无超时
  -- 时间戳
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  expires_at TEXT,
  -- 外键（软关联，SQLite 不强制 FK）
  -- workflow_job_id TEXT REFERENCES workflow_jobs(id)
);
```

> ⚠️ 迁移策略：在 builtin-sqlite.ts 中用 ALTER TABLE ADD COLUMN 逐列添加，
> 保留现有 `task_id` 列不删（重命名为 `resource_id` 的语义扩展需要数据迁移，建议用新列）。
