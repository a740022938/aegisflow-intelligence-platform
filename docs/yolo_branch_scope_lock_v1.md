# YOLO 支线范围锁定 V1

## 1. 文档目的
- 锁定 YOLO 十连发支线前期施工边界。
- 防止新助手在首轮接力时越界扩张。

## 2. 当前阶段定义
- 阶段名：`Pre-Construction / Blueprint + Minimal Start`
- 主线状态：`AGI Model Factory v6.5.0 封板基线`
- 支线状态：`独立规划与分期施工`

## 3. 当前允许范围（In Scope）
1. 文档与图纸输出：路线、边界、职责、契约、风险、接力包。
2. 五连发最小链路的接口预留（仅后续新助手实施）。
3. 审计留痕模板和运行结果记录模板。
4. 新增代码时优先局限在以下目录：
   - `apps/local-api/src/vision-branch/**`
   - `workers/vision-branch/**`
   - `packages/vision-contracts/**`
   - `docs/**`、`audit/**`

## 4. 当前禁止范围（Out of Scope for this round）
1. 不进行十连发完整落地。
2. 不做主线架构重构。
3. 不做大规模数据库 schema 改造。
4. 不做未经说明的主线 UI 重做。
5. 不新增重型依赖栈。
6. 不清理/删除现有主线历史资产。

## 5. 变更白名单
- 文件类型建议：`.md`, `.json`, `.ts`（接口/空实现）
- 优先改动：`docs/`, `audit/`
- 代码改动策略：`小步可审计 + 单任务单目的`

## 6. 阶段门禁
满足以下条件才允许进入下一施工阶段：
1. 五连发接口契约冻结。
2. 模块职责和边界无歧义。
3. 接力任务包已拆分并可独立验收。
4. 风险与审批点已写明。

## 7. 交接约束给新助手
- 先做基础链路闭环，再做增强模块。
- 所有新模块必须遵守统一输出契约。
- 任何自动训练/部署动作必须默认人工审批。

## 8. 版本标识建议
- 本蓝图文档版本：`V1`
- 后续施工建议版本节奏：
  - `v6.6.x`：五连发基础链
  - `v6.7.x`：八连发增强链
  - `v6.8.x`：十连发自动闭环
