## 1. Recovery Types

### A. Release Package Recovery
从已封存的 Release Package 恢复。

**适用场景**:
- 需要回滚到已发布版本
- 生产环境问题需要快速切换

**前置条件**:
- Release Package 存在
- Manifest 完整
- Backup 可用

**操作步骤**:
```bash
# 1. 验证目标 Release Package
python scripts/recovery_verify.py --target <release_id> --drill

# 2. 确认 Backup 状态
# 查看 audit_logs 中最近的 backup_created 记录

# 3. 执行 Recovery（手动）
# - 从 backup 恢复数据库文件
# - 或从 Release Package 重新部署
```

### B. Database Recovery
从 Backup 恢复数据库。

**适用场景**:
- 数据库损坏
- 数据丢失
- 需要回滚到历史状态

**前置条件**:
- Backup 文件存在
- Backup 未损坏

**操作步骤**:
```bash
# 1. 停止 API 服务
Stop-Process -Name tsx -Force

# 2. 执行 WAL checkpoint
python -c "import sqlite3; conn = sqlite3.connect('path/to/db'); conn.execute('PRAGMA wal_checkpoint(TRUNCATE)'); conn.close()"

# 3. 恢复 Backup
Expand-Archive -Path <backup.zip> -DestinationPath <restore_location>

# 4. 验证恢复
python scripts/recovery_verify.py

# 5. 重启 API 服务
```

## 2. Rollback Triggers

触发回滚的条件：

| Trigger | Condition | Action |
|---------|-----------|--------|
| Seal Verification Failed | recovery_verify.py 返回非零 | 阻止发布 |
| Post-Seal Metrics Degrade | 指标下降超过阈值 | 评估回滚 |
| Production Incident | 线上问题 | 执行 Recovery |
| Data Corruption Detected | 数据校验失败 | 执行 Database Recovery |

## 3. Recovery Audit Trail

所有恢复操作必须记录：

```sql
INSERT INTO recovery_logs (
  id, recovery_type, target_type, target_id, 
  status, source_backup, source_release,
  verification_json, notes, performed_by, performed_at
) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?);
```

**字段说明**:
- `recovery_type`: 'drill' / 'actual' / 'rollback'
- `status`: 'success' / 'failed' / 'partial'
- `verification_json`: 验证结果
- `performed_by`: 操作人

---

## 4. Recovery Drill Schedule

**建议频率**: 每周至少 1 次

**Drill 类型**:
1. Release Package Integrity Drill
2. Backup Restoration Drill（模拟）
3. End-to-End Recovery Drill（全链）

**Drill 命令**:
```bash
python scripts/recovery_verify.py --drill
```

---

## 5. Actions After Recovery

恢复后必须完成：

1. ✅ 验证数据一致性（regression_v500.py）
2. ✅ 验证主线 Gates（所有 Gate passed）
3. ✅ 更新 audit_logs（记录 recovery 事件）
4. ✅ 通知相关人员
5. ✅ Post-mortem 文档（如适用）
