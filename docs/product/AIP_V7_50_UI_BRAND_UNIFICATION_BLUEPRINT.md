### 4.2 Empty State Copy Standardization

**Problem**: 25+ empty state texts use inconsistent styles (emoji prefix, plain text, bilingual mix).

**Standard**: All empty states should follow one of two patterns:

| Pattern | Example | When |
|---------|---------|------|
| **Simple** | `暂无数据` / `No data` | Generic empty states |
| **Actionable** | `暂无知识条目。可点击"新增知识"创建第一条记录。` | Pages with a clear next-step CTA |

- Remove emoji prefixes from empty state text (emoji can stay in the `icon` prop of `EmptyState` component)
- Ensure all empty states have bilingual text (Chinese + English) where possible
- Standardize fallback in `EmptyState.tsx` component

## 6. Route-Level Feature Flag / Hidden Preview Safety Policy

### 6.1 Current Risk

- **40+ preview routes** are defined in `App.tsx` with **no** route-level feature flag or permission check
- **15+ Stage C routes** are live and functional despite `AIP_STAGE_C = 'disabled'` in metadata
- Any user who discovers the URL can access any preview route
- 5 readonly/debug pages are visible in the sidebar with confusing names (`-readonly` suffix exposed to users)

### 6.2 Route Safety Policy

| Principle | Rule |
|-----------|------|
| **All preview routes** MUST have a feature flag or gate check before rendering | Enforced by route wrapper or middleware |
| **Stage C routes** MUST return 404 or redirect when `AIP_STAGE_C === 'disabled'` | Enforced regardless of direct URL access |
| **Hidden direct routes** are acceptable IF they have a safety boundary | Must be documented in registry |
| **Sidebar-visible routes** must NOT contain `-readonly`, `-preview`, `-debug` in their display label | User-facing label should be clean |

### 6.3 Implementation Plan

| # | Action | Protection Level | Effort |
|---|--------|-----------------|--------|
| 1 | Create `<RouteGuard>` component that checks `FEATURE_FLAGS.STAGE_C_ENABLED` before rendering Stage C routes | Stage C routes | Medium |
| 2 | Wrap all preview routes with a `process.env.NODE_ENV === 'development'` guard or an explicit `AIP_ALLOW_PREVIEW_ROUTES` flag | Preview routes | Medium |
| 3 | Create a route metadata registry that maps each route to its exposure level and required gate | All routes | Large |
| 4 | Rename sidebar NavItem labels to remove `readonly`/`debug` suffixes (e.g., "Advanced Mode Preview" → "Advanced Mode"; "Mahjong Debug" → "Mahjong Lab") | Sidebar | Small |
| 5 | Document all hidden routes in `navigation-exposure-registry.ts` with `allowedNow: false` where applicable | Registry | Medium |
