## 1. 本版做什么

### A. 主线 Gate 化
对现有主线阶段建立统一 Gate 机制：

| Gate 名称 | 对应阶段 | 已有能力 | 本版补充 |
|-----------|----------|----------|----------|
| Evaluation Ready Gate | Evaluation 完成 | status 字段 | Gate 定义 + checks + blocking |
| Artifact Ready Gate | Artifact 产出 | status + source_type | Gate 定义 + readiness 扩展 |
| Promotion Ready Gate | 晋升候选 | promotion_status + readiness | 已有，需统一格式 |
| Release Ready Gate | 发布就绪 | release record + approval | Gate 定义 + seal pre-check |
| Seal Ready Gate | 封存就绪 | sealed status + manifest | Gate 定义 + backup check |

每个 Gate 统一包含：
- `gate_name`: Gate 标识
- `stage_name`: 对应阶段
- `required_inputs`: 必须输入
- `required_checks`: 必须检查项
- `fail_reasons`: 失败原因列表
- `pass_result`: 通过结果
- `audit_record`: 审计记录 ID
- `blocking_status`: 阻塞状态

### B. 工厂运行态视图
新建 `/factory-status` 页面或 Dashboard 增强，聚合展示：
- 主线阶段状态概览
- Gate 阻塞点列表
- 审批挂起点列表
- 最近 24h 失败任务
- 失败原因聚合
- 最近发布状态
- 最近封板状态
- 最近备份状态

### C. 恢复 / 回滚基线
基于现有 Release Package / Seal Manifest / Backup：
- 恢复校验流程（脚本）
- 回滚触发条件定义
- 恢复/回滚审计留痕
- 恢复演练记录模板

### D. 主线回归基线
建立固定验证入口：
- Golden Path 冒烟脚本（完整主线）
- 失败注入验证脚本
- Gate 拦截验证脚本
- Backup/Seal/Release 一致性校验脚本

## 2. 本版允许的技术改动范围

### Schema 变更（仅限增量）
允许：
- 新建 `gate_checks` 表（记录 Gate 检查结果）
- artifacts/evaluations/models 等表增加 `gate_status` 列（若需要）
- 新建 `recovery_logs` 表（恢复演练记录）

禁止：
- 删除任何现有列
- 修改任何现有列类型
- 重命名任何现有表/列
- 全表迁移

### API 变更（仅限增量）
允许：
- 新增 `/api/gate/*` 系列端点
- 新增 `/api/factory/status` 聚合端点
- 新增 `/api/recovery/*` 系列端点

禁止：
- 删除任何现有端点
- 修改现有端点的请求/响应格式（可扩展，不可破坏）

### UI 变更（仅限增量）
允许：
- 新建 `/factory-status` 页面
- 现有页面增加 Gate 状态展示
- 现有页面增加恢复/回滚操作入口

禁止：
- 重做任何现有页面布局
- 大规模更换 UI 组件库
- 改变现有页面路由结构
