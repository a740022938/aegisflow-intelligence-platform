## 1. 本版做什么

### A. Health Patrol 视图
新增健康巡检视图/面板，聚合以下信息：

- **最近一次 Regression 结果**: 最后执行 regression_v500.py 的结果
- **最近一次 Recovery Drill 结果**: recovery_logs 最新记录
- **最近一次 Release Validation 结果**: gate 检查和 validation 摘要
- **当前 Blocked Gates 数**: gate_checks WHERE status='blocked'
- **当前 Active Incidents 数**: incident 计数（通过 audit 判断）
- **最近失败趋势**: workflow_jobs WHERE status='failed'
- **当前 Rollback Readiness**: 复用 v5.3.0 逻辑
- **当前 Backup 健康状态**: backup_verified 和 package_present

目标：
让用户一眼看到"工厂今天健不健康"。

### B. 持续验证结果聚合
统一聚合以下验证结果：

| 验证类型 | 数据来源 | 显示字段 |
|----------|----------|----------|
| Regression | regression_v500 执行记录 | 通过/失败、时间 |
| Recovery Drill | recovery_logs | 成功/失败、演练时间 |
| Release Validation | gate_checks + releases | Gate 摘要 |
| Backup Consistency | releases.backup_verified | 验证状态 |
| Seal Consistency | releases.package_present | 封板状态 |

目标：
验证不再散落，形成"持续验证总览"。

### C. 趋势统计与时间窗口对比
支持以下趋势维度：

- **失败趋势**: 24h / 7d / 30d workflow_jobs 失败数
- **Blocked Gates 趋势**: 各时间窗口 blocked gate 数
- **Recovery Drill 成功率**: 成功次数 / 总次数
- **Rollback Readiness 分布**: ready / caution / blocked

目标：
不是只看一个点，而是能看"变好还是变坏"。

### D. 风险前置提示
基于以下规则显示风险状态：

| 规则 | 条件 | 状态 |
|------|------|------|
| Recovery 过久未演练 | >30 天无成功 drill | caution |
| Regression 过久未执行 | >7 天无执行 | caution |
| Blocked Gates 增长趋势 | 7d blocked > 24h blocked | warning |
| Active Incidents 过多 | >5 个活跃 incident | blocked |
| Backup 状态异常 | backup_verified=0 | blocked |
| Rollback 非 ready | rollback readiness ≠ ready | caution |

输出：
- **healthy**: 无风险
- **caution**: 有轻微风险
- **warning**: 有明显风险
- **blocked**: 存在阻塞问题

### E. 巡检结论与运维入口闭环
建立以下跳转链路：

- Health Patrol → Factory Status
- Health Patrol → Release Governance
- Health Patrol → Audit / WorkflowJobs
- Health Patrol → Incident / Timeline
- Health Patrol → Backup / Recovery Detail
