## 1. 本版做什么

### A. Gate Drilldown
对 Factory Status 中的 Blocked Gates 增加可点击下钻能力。

目标：
用户点击某个 blocked gate → 看到该 gate 下所有阻塞记录的详情。

下钻内容包括：
- gate_name / stage_name
- 阻塞数量
- 失败原因列表
- 关联实体列表（entity_id 列表）
- 最近阻塞时间
- 可跳转相关审计

实现方式：
- 新增 `GET /api/gates/drilldown?gate_name=xxx&status=blocked&limit=50`
- FactoryStatus 页面增加 Gate 详情面板（可展开抽屉）

### B. 失败追踪能力
对 Factory Status 中的"失败原因聚合"和"最近失败任务"增加可点击追踪能力。

目标：
点击失败原因 → 看到关联任务列表
点击任务 → 看到详细上下文

实现方式：
- 扩展 `GET /api/factory/status` 支持 `?time_range=24h|7d|30d`
- 新增 `GET /api/factory/failures?time_range=24h&limit=50` 返回失败任务列表含 error_message
- FactoryStatus 增加 Failure 详情面板

### C. Backup / Recovery 详情能力
对 Factory Status 中的备份状态和恢复演练增加详情查看。

目标：
查看最近备份产物路径、checksum、manifest 摘要、recovery drill 结果。

实现方式：
- `GET /api/factory/status` 扩展返回 backup_verified（基于真实备份产物）
- FactoryStatus 增加 Backup Detail 面板
- Recovery Drill 列表增加 Drill ID / Status / Notes

### D. 时间过滤能力
Factory Status 增加时间范围切换。

目标：
24h / 7d / 30d 切换控制 recent_failures、recent_releases、blocked_gates 的时间范围。

实现方式：
- `GET /api/factory/status?time_range=24h` 支持时间过滤参数
- 前端时间切换控件

### E. 审计跳转能力
Factory Status 中以下区域支持下钻跳转：
- Gate → Audit（关联 gate_checks 记录）
- Recent Failures → Audit（关联 workflow_jobs）
- Recent Releases → Audit（关联 audit_logs）
- Recovery Drills → Audit（关联 recovery_logs）

实现方式：
- 各详情面板增加"查看审计"按钮
- 使用 React Router Link 跳转
