- **状态**: 冻结

## 1. 数据来源说明

### 1.1 合法数据来源

- **来源 A**：人工拍摄（手机/相机）麻将牌桌面照片，JPG/PNG，单张包含 1~16 张麻将牌
- **来源 B**：视频截帧，经抽帧后转为图像
- **来源 C**：已有的标注数据集（需确认版权和标注格式）

### 1.2 禁止使用

- 含个人信息（人脸、车牌等）的图像
- 分辨率低于 640×480 的图像
- 严重遮挡（>30% 面积）的牌面

### 2.2 标签文件格式（YOLO Detection）

```
<class_id> <x_center> <y_center> <width> <height>
```

- 所有坐标归一化到 [0, 1]
- 每行一条标注
- 多标签用多行表示
- 文件名为对应图像文件名（不含扩展名）+ `.txt`

**示例**（图像 1920×1080，检测到 3 张牌）：
```
5 0.521 0.485 0.062 0.078
12 0.634 0.501 0.058 0.075
27 0.782 0.493 0.055 0.072
```

## 3. 文件命名规范

### 3.1 raw 层命名

```
{YYYYMMDD}_{seq}_{source}.{ext}
```

示例：
- `20260415_001_cam1.jpg`
- `20260415_002_video01.png`

### 3.2 clean 层命名

```
{source_id}_{seq4padded}.{ext}
```

示例：
- `raw001_0001.jpg`
- `raw001_0002.jpg`

**规则**：
- 图像和标签文件**同名**（仅扩展名不同）
- `seq` 从 0001 开始，不补零则后续排序会出现 1, 10, 11...
- 禁止在文件名中出现空格、中文、特殊字符

## 4. 图像/标签一致性规则

### 4.1 一致性检查

每张图像必须对应一个同名的标签文件（.txt），即使无标注也要有空文件。

检查规则：
- `image_count == label_count`（允许空标签文件）
- 图像分辨率宽高比在 [0.5, 2.0] 范围内
- 每张图像至少包含 1 个标注框（clean 层，test 集除外）

### 4.2 坏图检测规则

| 检查项 | 阈值 | 处理方式 |
|--------|------|----------|
| 分辨率过低 | width < 640 或 height < 480 | 移入坏图目录 |
| 文件损坏 | 无法解码 | 移入坏图目录 |
| 标注越界 | 任意坐标 > 1.0 或 < 0.0 | 修正或丢弃该标注 |
| class_id 超范围 | id < 0 或 id > 33 | 丢弃该标注行 |
| 标注框过小 | area < 0.001（相对图像面积） | 丢弃该标注行 |
| 标注框超出图像 | 坐标越界 | 修正归一化坐标 |

坏图目录：`E:\AGI_Factory\data\mahjong\bad\`
