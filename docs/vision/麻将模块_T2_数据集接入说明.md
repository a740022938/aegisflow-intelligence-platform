### 2.2 raw 层规则

- **只读**：raw 层任何时候不得被覆盖或修改
- **来源隔离**：不同采集来源放入不同 `source_id` 子目录
- **命名**：原始图像文件名保持采集时的原始命名，不强制重命名
- **元数据**：raw 层每个 source_id 必须包含 `_meta.json` 说明采集信息（时间、设备、环境）

## 4. 版本治理规则

### 4.1 dataset_version 命名规范

格式：`{dataset_code}_{YYYYMMDD}_{seq}`

示例：
- `mahjong_v1_20260415_001` — 麻将数据集，2026-04-15 第一次清洗版本
- `mahjong_v1_20260420_002` — 同一数据集，2026-04-20 第二次清洗版本（增补数据后）

版本命名由 Datasets API 写入 `datasets` 表的 `version` 字段。

### 4.2 版本不可覆盖原则

- `version` 字段一旦写入 `datasets` 表，**不修改**
- 新版本使用新的 `version` 值
- 训练任务必须绑定具体 `version`，禁止用 "latest"

### 4.3 split 比例规范

| split | 比例 | 最小样本数 |
|-------|------|-----------|
| train | 80% | 400 |
| val | 15% | 75 |
| test | 5% | 25 |

**说明**：总样本 ≥500 张时使用上述比例；若不足 500 张，test 集至少 5 张，其余归 train。

## 5. 与训练任务的绑定规则

### 5.1 绑定字段

训练任务创建时，通过以下字段绑定数据集：

| 字段 | 位置 | 说明 |
|------|------|------|
| dataset_id | training_config 或 run config_json | datasets 表主键 |
| dataset_version | config_json.params_json | dataset_version 字符串 |
| dataset_path | config_json.resource_json | 数据集根目录 |

### 5.2 绑定验证（受控接入）

创建训练任务前，验证：
1. dataset_id 存在且 status='active'
2. dataset_version 在 datasets 表中有记录
3. split manifest 文件存在于 `split/{version}/` 目录
