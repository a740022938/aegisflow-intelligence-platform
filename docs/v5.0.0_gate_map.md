# v5.0.0 Gate Map — 主线阶段门禁映射

**Date**: 2026-04-13  
**Version**: 5.0.0

---

## Gate 定义

每个 Gate 统一结构：

```typescript
interface GateResult {
  gate_name: string;        // Gate 标识
  stage_name: string;       // 对应阶段
  entity_id: string;        // 实体 ID
  entity_type: string;      // 实体类型
  status: 'passed' | 'blocked' | 'pending' | 'failed';
  required_inputs: string[];  // 必须输入
  required_checks: string[];  // 必须检查项
  check_results: { check: string; passed: boolean; reason?: string }[];
  fail_reasons: string[];     // 失败原因列表
  pass_result?: string;       // 通过结果描述
  audit_record: string;       // 审计记录 ID
  blocking_status?: string;   // 阻塞状态详情
  checked_at: string;         // 检查时间
}
```

## 1. Evaluation Ready Gate

**阶段**: Evaluation 完成  
**触发点**: Evaluation status → completed  
**实体**: evaluations

### Required Inputs
- evaluation_id
- model_id (or model_family)
- dataset_id

### Required Checks
| Check | 条件 | 失败原因 |
|-------|------|----------|
| status_completed | status === 'completed' | Evaluation not completed |
| has_model | model_id 或 model_name 非空 | No model linked |
| has_dataset | dataset_id 非空 | No dataset linked |
| has_metrics | result_summary_json 非空 | No metrics recorded |
| no_error | error_message 为空 | Evaluation has error |

### Pass Result
- 可以生成 Artifact
- 可以进入下一阶段

### Blocking Status
- `model_missing`: 缺少模型关联
- `dataset_missing`: 缺少数据集关联
- `metrics_missing`: 缺少指标记录
- `error_state`: 评估处于错误状态

---

## 2. Artifact Ready Gate

**阶段**: Artifact 产出  
**触发点**: Artifact 创建后  
**实体**: artifacts

### Required Inputs
- artifact_id
- source_type
- evaluation_id (若 source_type=evaluation)

### Required Checks
| Check | 条件 | 失败原因 |
|-------|------|----------|
| has_path | path/storage_path 非空 | No storage path |
| has_model_family | model_family 非空 | No model family |
| has_format | format 非空 | No format specified |
| status_ready | status === 'ready' | Artifact not ready |
| source_valid | source_type in allowed values | Invalid source type |
| evaluation_valid | 若 source_type=evaluation，则 evaluation_id 非空 | Missing source evaluation |

### Pass Result
- 可以进入 Promotion Ready Gate
- 可以开始晋升流程

### Blocking Status
- `storage_missing`: 缺少存储路径
- `metadata_incomplete`: 缺少关键元数据
- `source_invalid`: 来源验证失败

---

## 3. Promotion Ready Gate

**阶段**: 晋升候选  
**触发点**: 请求 promote 时  
**实体**: artifacts

### Required Inputs
- artifact_id
- evaluation_id
- model_family

### Required Checks
| Check | 条件 | 失败原因 |
|-------|------|----------|
| status_approved | promotion_status in ['draft', 'rejected'] | Already promoted |
| has_evaluation | evaluation_id 非空 | No linked evaluation |
| has_metrics | metrics_snapshot_json 非空 | No metrics snapshot |
| has_model_family | model_family 非空 | No model family |
| has_format | format 非空 | No format specified |

### Pass Result
- promotion_status → candidate 或 approval_required
- 创建 Approval 记录（若需要审批）

### Blocking Status
- `already_promoted`: 已晋升
- `no_evaluation`: 缺少来源评估
- `no_metrics`: 缺少指标快照
- `metadata_incomplete`: 元数据不完整

**注**: 此 Gate 已在 v4.8.0 实现（getPromotionReadiness），本版统一格式。

---

## 4. Release Ready Gate

**阶段**: 发布就绪  
**触发点**: Approval 通过后，准备 seal  
**实体**: artifacts (approved 状态)

### Required Inputs
- artifact_id
- approval_id
- model_id

### Required Checks
| Check | 条件 | 失败原因 |
|-------|------|----------|
| status_approved | promotion_status === 'approved' | Not approved |
| has_approval | approval_id 非空且 status='approved' | No approval record |
| has_model | model_id 已解析 | Cannot resolve target model |
| not_sealed | release_id 为空 | Already sealed |
| backup_exists | 最近备份记录存在 | No backup record |

### Pass Result
- 可以执行 Seal Release
- 创建 Release 记录

### Blocking Status
- `not_approved`: 未通过审批
- `approval_missing`: 缺少审批记录
- `model_unresolved`: 无法解析目标模型
- `already_sealed`: 已封存
- `no_backup`: 缺少备份

---

## 5. Seal Ready Gate

**阶段**: 封存就绪  
**触发点**: Seal Release 执行后  
**实体**: releases

### Required Inputs
- release_id
- artifact_id
- manifest_json

### Required Checks
| Check | 条件 | 失败原因 |
|-------|------|----------|
| manifest_valid | release_manifest_json 解析成功 | Invalid manifest |
| has_lineage | manifest.lineage 非空 | No lineage recorded |
| has_metrics | metrics_snapshot_json 非空 | No metrics snapshot |
| has_notes | release_notes 非空 | No release notes |
| audit_logged | audit_logs 有 seal 事件 | No audit record |

### Pass Result
- release.status → sealed
- 可以进行恢复演练

### Blocking Status
- `manifest_invalid`: 清单无效
- `lineage_missing`: 缺少链路记录
- `notes_missing`: 缺少发布说明
- `audit_missing`: 缺少审计记录

---

## Gate 检查流程

```
Evaluation → [Evaluation Ready Gate]
    ↓ passed
Artifact → [Artifact Ready Gate]
    ↓ passed
Promotion → [Promotion Ready Gate]
    ↓ passed
Approval → [Release Ready Gate]
    ↓ passed
Seal → [Seal Ready Gate]
    ↓ passed
SEALED RELEASE
```

## Gate 记录存储

新建 `gate_checks` 表：

```sql
CREATE TABLE IF NOT EXISTS gate_checks (
  id TEXT PRIMARY KEY,
  gate_name TEXT NOT NULL,
  stage_name TEXT NOT NULL,
  entity_id TEXT NOT NULL,
  entity_type TEXT NOT NULL,
  status TEXT NOT NULL,  -- passed/blocked/pending/failed
  check_results_json TEXT NOT NULL,
  fail_reasons_json TEXT,
  pass_result TEXT,
  audit_record TEXT,
  blocking_status TEXT,
  checked_at TEXT NOT NULL,
  created_at TEXT NOT NULL
)
```
