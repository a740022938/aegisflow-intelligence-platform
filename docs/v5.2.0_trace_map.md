## 2. 关联链路设计

### A. Gate -> Entity
| Source | Target | Join Key |
|--------|--------|----------|
| gate_checks.entity_id | artifacts.id | entity_id |
| gate_checks.entity_id | evaluations.id | entity_id |
| gate_checks.entity_id | models.id | entity_id |
| gate_checks.entity_id | releases.id | entity_id |

**API**: `GET /api/correlate?gate_check_id=xxx`

**Returns**:
```json
{
  "gate_check": { "id": "...", "gate_name": "promotion_ready", "status": "blocked" },
  "entity": { "type": "artifact", "id": "...", "promotion_status": "draft" },
  "related_job": { "id": "...", "name": "eval-model-v410-..." },
  "related_audit": [ { "id": "...", "action": "gate_checked" } ]
}
```

### B. Job -> Steps -> Error
| Source | Target | Join Key |
|--------|--------|----------|
| workflow_jobs.id | job_steps.job_id | job_id |
| job_steps.error_message | — | 直接字段 |

**API**: `GET /api/timeline?job_id=xxx`

**Returns**: 按时间排序的 job_step 状态变化事件

### C. Release -> Artifact -> Backup -> Recovery
| Source | Target | Join Key |
|--------|--------|----------|
| releases.artifact_id | artifacts.id | artifact_id |
| releases.id | recovery_logs.source_release | source_release |
| recovery_logs.source_backup | backup path | — |

**API**: `GET /api/correlate?release_id=xxx`

**Returns**:
```json
{
  "release": { ... },
  "artifact": { ... },
  "backup": { "path": "...", "verified": true },
  "recovery_drills": [ { "id": "...", "status": "success" } ]
}
```

### D. Recovery -> Audit -> Artifact
| Source | Target | Join Key |
|--------|--------|----------|
| recovery_logs.target_id | artifacts.id | target_id |
| audit_logs.target | recovery_logs.id | target LIKE '%recovery%' |

**API**: `GET /api/correlate?recovery_id=xxx`

## 3. Timeline 设计

### 事件来源优先级（按时间排序）
1. gate_checks.checked_at — Gate 状态变化
2. job_steps.updated_at — 步骤状态变化
3. audit_logs.created_at — 审计事件
4. releases.sealed_at — 发布事件
5. recovery_logs.performed_at — 恢复演练事件

### Timeline API
```
GET /api/timeline?entity_type=artifact&entity_id=xxx
GET /api/timeline?job_id=xxx
GET /api/timeline?gate_check_id=xxx
GET /api/timeline?release_id=xxx
```

### 统一事件结构
```typescript
interface TimelineEvent {
  timestamp: string;      // ISO 时间
  event_type: 'gate_check' | 'job_step' | 'audit' | 'release' | 'recovery';
  event_id: string;
  entity_type: string;
  entity_id: string;
  action: string;         // 'status_changed' | 'error' | 'sealed' | 'drill'
  summary: string;
  status?: string;
  error_message?: string;
  related_to: Array<{ type: string; id: string }>;
}
```

### Timeline SQL 示例
```sql
-- Gate events
SELECT 
  checked_at as timestamp,
  'gate_check' as event_type,
  id as event_id,
  entity_type,
  entity_id,
  'status_changed' as action,
  gate_name || ' ' || status as summary,
  status
FROM gate_checks
WHERE entity_id = ? OR id = ?
ORDER BY checked_at DESC;

-- Job step events
SELECT 
  updated_at as timestamp,
  'job_step' as event_type,
  id as event_id,
  'workflow_job' as entity_type,
  job_id as entity_id,
  'status_changed' as action,
  step_key || ' ' || status as summary,
  status,
  error_message
FROM job_steps
WHERE job_id = ?
ORDER BY updated_at DESC;

-- Audit events
SELECT 
  created_at as timestamp,
  'audit' as event_type,
  id as event_id,
  category as entity_type,
  target as entity_id,
  action,
  action || ' on ' || target as summary
FROM audit_logs
WHERE target LIKE ? OR detail_json LIKE ?
ORDER BY created_at DESC;
```

## 4. Incident Drilldown 设计

### Incident 来源
- 最近失败任务（workflow_jobs.status='failed'）
- 阻塞 Gate（gate_checks.status='blocked'）
- 发布异常（releases.package_present=0 OR backup_verified=0）
- 恢复失败（recovery_logs.status='failed'）

### Incident API
```
GET /api/incident?id=xxx&type=failure|gate_block|release_issue
```

### Incident 结构
```typescript
interface Incident {
  id: string;
  incident_type: 'failure' | 'gate_block' | 'release_issue';
  root_entity_type: string;
  root_entity_id: string;
  root_cause_step?: string;
  related_job_id?: string;
  related_gate_check_id?: string;
  related_release_id?: string;
  error_message?: string;
  first_seen_at: string;
  last_seen_at: string;
  latest_status: string;
  timeline_url: string;
  audit_url: string;
}
```

### Incident SQL 示例
```sql
-- 失败任务 -> 首个失败步骤
SELECT 
  j.id,
  j.name,
  j.error_message,
  j.updated_at as last_seen_at,
  s.step_key as root_cause_step,
  s.started_at as first_seen_at,
  s.status as step_status
FROM workflow_jobs j
LEFT JOIN job_steps s ON j.id = s.job_id AND s.status = 'failed'
WHERE j.status = 'failed' AND j.id = ?
ORDER BY s.started_at ASC LIMIT 1;
```

## 5. 过滤与区分设计

### version_prefix 过滤
基于对象 ID 或名称前缀：
- artifact: `artifact-v510-...`
- evaluation: `eval-model-v510-...`
- release: name 前缀

**SQL**:
```sql
WHERE name LIKE 'v5%' OR id LIKE '%-v5-%'
```

### active_only 过滤
- workflow_jobs: status IN ('running', 'paused', 'blocked')
- gate_checks: blocking_status IS NOT NULL
- releases: status != 'archived'

### 时间范围
已有实现，复用 v5.1.0 的 `time_range` 参数。

## 7. 导航闭环设计

### 参数传递
- Factory Status → Timeline: `?entity_type=artifact&entity_id=xxx&time_range=7d`
- Timeline → Audit: `/audit?target=xxx&time_range=7d`
- Audit → Factory Status: `/factory-status?time_range=7d&highlight=xxx`

### 返回路径
每个详情页保留 query 参数：
```tsx
const navigate = useNavigate();
const returnTo = `/factory-status?time_range=${timeRange}`;
navigate(`/timeline?entity_id=${id}&return_to=${encodeURIComponent(returnTo)}`);
```
