### B. 数据源 SQL

#### Regression 结果
```sql
-- 从 audit_logs 查找最近 regression 执行
SELECT created_at, result, detail_json 
FROM audit_logs 
WHERE category = 'system' AND action LIKE '%regression%'
ORDER BY created_at DESC LIMIT 1;
```

#### Recovery Drill 结果
```sql
SELECT id, status, performed_at 
FROM recovery_logs 
WHERE recovery_type = 'drill' 
ORDER BY performed_at DESC LIMIT 1;
```

#### Gate 汇总
```sql
SELECT status, COUNT(*) as c 
FROM gate_checks 
WHERE checked_at > datetime('now', '-7 days')
GROUP BY status;
```

#### 失败趋势
```sql
SELECT 
  COUNT(CASE WHEN updated_at > datetime('now', '-24 hours') THEN 1 END) as failures_24h,
  COUNT(CASE WHEN updated_at > datetime('now', '-7 days') THEN 1 END) as failures_7d,
  COUNT(CASE WHEN updated_at > datetime('now', '-30 days') THEN 1 END) as failures_30d
FROM workflow_jobs 
WHERE status = 'failed';
```

## 3. 风险判断规则

### 规则表

| 规则 | 条件 | 状态 | 优先级 |
|------|------|------|--------|
| recovery_drill_age | >30 天无成功 drill | caution | 中 |
| regression_age | >7 天无 regression 执行 | caution | 高 |
| blocked_gates_trend | 7d blocked > 24h blocked * 1.5 | warning | 高 |
| active_incidents | >5 个活跃 incident | blocked | 最高 |
| backup_health | backup_verified=0 | blocked | 最高 |
| rollback_readiness | rollback readiness ≠ ready | caution | 中 |

### 整体状态计算

```
if any rule status === 'blocked':
  overall_status = 'blocked'
else if any rule status === 'warning':
  overall_status = 'warning'
else if any rule status === 'caution':
  overall_status = 'caution'
else:
  overall_status = 'healthy'
```

## 6. SQL 查询示例

### 失败趋势
```sql
-- 按天统计失败数
SELECT date(updated_at) as day, COUNT(*) as c 
FROM workflow_jobs 
WHERE status = 'failed' 
  AND updated_at > datetime('now', '-30 days')
GROUP BY day 
ORDER BY day DESC;
```

### Blocked Gates 趋势
```sql
SELECT date(checked_at) as day, COUNT(*) as c 
FROM gate_checks 
WHERE status = 'blocked' 
  AND checked_at > datetime('now', '-30 days')
GROUP BY day 
ORDER BY day DESC;
```

### Recovery 成功率
```sql
SELECT 
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) * 1.0 / COUNT(*) as rate
FROM recovery_logs 
WHERE performed_at > datetime('now', '-30 days');
```
