## 1. 本版做什么

### A. 安装/初始化/启停/升级/恢复文档收口
形成单机可执行文档，覆盖：

- **环境基线**: Node.js 22.x, Python 3.11+, SQLite 3
- **初始化步骤**: 数据库初始化、依赖安装、配置检查
- **启动步骤**: API server + Vite dev server
- **停止步骤**: 优雅关闭、端口释放
- **升级步骤**: git pull + npm install + restart
- **恢复步骤**: 从备份恢复的完整流程
- **故障排查**: 常见问题与解决入口
- **FAQ**: 高频问题与答案

目标：换人、换天、换窗口，仍能顺利拉起系统。

### B. 单机运维总入口/总说明
建立运维主索引，覆盖：

- Factory Status（工厂状态）
- Release Governance（发布治理）
- Health Patrol（健康巡检）
- Audit（审计追踪）
- Workflow Jobs（任务列表）
- Backup/Recovery（备份恢复）
- Regression/Verify（回归验证）

目标：让使用者有明确运维入口，不用到处找。

### C. 迁移/备份/恢复标准作业说明
形成标准作业文档，覆盖：

- 备份产物清单（DB snapshot、Seal zip、Manifest）
- Seal/Backup/DB Snapshot 对应关系
- 恢复前检查项
- 恢复操作步骤
- 恢复后验证项
- 迁移需带走的关键目录
- 迁移后必跑验证

目标：系统能"按标准动作操作"。

### D. P3 验收清单与验收包
输出条目化验收材料：

- P3_acceptance_checklist.md（验收清单）
- P3_readiness_matrix.md（就绪矩阵）
- P3_demo_path.md（演示路径）
- P3_risk_register.md（风险登记册）
- P3_closure_report.md（收口报告）

目标：把 P3 从"感觉差不多了"变成"条目化可验收"。

### E. 版本交接资料
输出交接文档：

- 当前版本能力总览
- 关键入口索引
- 文档索引
- 脚本索引
- 备份位置
- 恢复方式
- 回归方式
- 后续 OpenClaw 接入前置条件

目标：具备"可交接性"。

### F. P3 正式封板前置条件清单
输出封板前置检查：

- 已满足项

- 硬门槛项

目标：为 P3 Official Seal 做准备。
