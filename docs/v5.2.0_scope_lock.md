## 1. 本版做什么

### A. Root Cause Timeline
为失败任务、阻塞 Gate、恢复失败、发布异常等对象建立统一时间线视图。

目标：
从一个失败点，能看到"前面发生了什么、后面发生了什么"。

实现方式：
- 新增 `GET /api/timeline?entity_id=xxx` 或 `GET /api/timeline?gate_check_id=xxx`
- 返回按时间排序的事件列表：
  - gate_check 事件（status 变化）
  - job_step 事件（status 变化、error）
  - audit 事件（相关操作）
  - release 事件（seal、backup）
  - recovery 事件（drill、actual）

事件统一结构：
```typescript
{
  timestamp: string;      // ISO 时间
  event_type: string;     // 'gate_check' | 'job_step' | 'audit' | 'release' | 'recovery'
  event_id: string;       // 原始事件 ID
  entity_type: string;    // 'workflow_job' | 'artifact' | 'release' | ...
  entity_id: string;
  action: string;         // 'status_changed' | 'error' | 'sealed' | ...
  summary: string;        // 人类可读摘要
  status?: string;        // 新状态
  related_to?: {         // 关联对象
    type: string; id: string;
  }[];
}
```

### B. Incident Drilldown
为"失败任务""阻塞 Gate"等建立统一故障详情视图。

实现方式：
- 新增 `GET /api/incident?id=xxx&type=failure|gate_block|release_issue`
- 返回聚合信息：
  - root_entity_type / root_entity_id
  - root_cause_step（首个失败步骤）
  - related_job_id / related_gate_check_id
  - timeline 入口
  - audit 入口

### C. 跨对象关联查询
打通以下链路：
- Gate -> Entity（artifact/evaluation/model/release）
- Job -> Steps -> Error
- Release -> Artifact -> Backup -> Recovery
- Recovery -> Audit -> Artifact

实现方式：
- 扩展现有 API 返回 related_to 字段
- 新增 `GET /api/correlate?entity_type=xxx&entity_id=xxx`

### D. 当前/历史区分
在 Factory Status 和相关页面增加：
- version_prefix 过滤（如 v5 / v4）
- active_only 过滤
- 时间范围过滤（已有，复用）

实现方式：
- 扩展现有 API 支持 `?version_prefix=v5&active_only=true`

### E. 审计联动导航闭环
确保页面跳转时保留上下文：
- time_range 参数传递
- entity_id 参数传递
- 返回链接（返回 Factory Status）
