## Overview

把主线从 `Evaluation -> Artifact` 推进到 `Artifact -> Promotion / Release Candidate -> Approval -> Audit -> Model Registry`。

## Schema Changes (incremental only)

### artifacts table
| Column | Type | Default | Purpose |
|--------|------|---------|---------|
| promotion_status | TEXT NOT NULL | 'draft' | draft/candidate/approval_required/approved/rejected/archived |
| promotion_comment | TEXT | '' | 审批备注 |
| approved_by | TEXT | '' | 审批人 |
| approved_at | TEXT | '' | 审批时间 |

### models table
| Column | Type | Default | Purpose |
|--------|------|---------|---------|
| promotion_status | TEXT NOT NULL | 'draft' | 发布候选状态 |
| source_artifact_id | TEXT | '' | 晋升来源 artifact |
| promotion_comment | TEXT | '' | 晋升备注 |

## New API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/artifacts/:id/promotion-readiness | 返回 promotion readiness score |
| POST | /api/artifacts/:id/promote | 晋升为候选（可选审批） |
| POST | /api/artifacts/:id/approve-promotion | 批准晋升 |
| POST | /api/artifacts/:id/reject-promotion | 拒绝晋升 |

## Promotion Flow

```
draft → [promote] → approval_required → [approve] → approved
                                              ↓
                                         [reject] → rejected → [re-promote] → approval_required
```

### Promotion Side Effects

1. `require_approval=true` (default):
   - artifact.promotion_status → approval_required
   - 创建 approval 记录 (action=promotion_review, resource_type=artifact)
   - 创建 audit 记录

2. `require_approval=false`:
   - artifact.promotion_status → candidate
   - model.promotion_status → candidate
   - model.source_artifact_id → artifact.id
   - 创建 audit 记录

3. Approve:
   - artifact.promotion_status → approved
   - model.promotion_status → approved
   - model.source_artifact_id → artifact.id
   - approval.status → approved
   - 创建 audit 记录

4. Reject:
   - artifact.promotion_status → rejected
   - approval.status → rejected
   - 创建 audit 记录
