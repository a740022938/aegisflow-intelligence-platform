- Deliver a minimal closed loop for cost routing.
- Route task types to specific execution routes with auditable reasons.

## 4. Route Resolution Logic (v1)
- Exact match: active policy by `task_type`, highest `priority` wins.
- Fallback match: `task_type='*'`, highest `priority` wins.
- Final fallback: if no policy found, route to `local_balanced`.
- Decision is persisted with:
  - `task_type`
  - `task_id` (optional)
  - `policy_id` (if matched)
  - `route_type`
  - `route_reason`
  - `input_json`
