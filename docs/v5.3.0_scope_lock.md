## 1. 本版做什么

### A. Release Governance 视图
新增发布治理视图/面板，聚合以下信息：

- **当前 Stable Release**: 最新已封存的稳定发布
- **当前 Release Candidate**: 最新候选发布（未封存）
- **最近发布记录**: 最近 5 条发布历史
- **最近回滚/恢复记录**: 最近恢复演练或实际回滚
- **发布前校验状态**: Gate 通过情况摘要
- **发布后验证状态**: Smoke / validation 结果
- **回滚就绪状态**: 是否具备回滚条件

目标：
让用户一眼看懂当前"发布态势"。

### B. Candidate vs Stable 对比能力
提供候选版与稳定版的对比视图，至少包含：

| 维度 | Candidate | Stable |
|------|-----------|--------|
| version | 显示 | 显示 |
| source_artifact | 显示 | 显示 |
| source_model | 显示 | 显示 |
| gate_status | 聚合 | 聚合 |
| seal_status | 未封存 | 已封存 |
| backup_status | 无 | 已验证 |
| validation_status | 显示 | 显示 |
| incident_count | 显示 | 显示 |
| created_at | 显示 | 显示 |
| sealed_at | 无 | 显示 |

目标：
不是只知道"有候选版"，而是能知道"它和当前稳定版差在哪"。

### C. Rollback Readiness 判断能力
提供系统可见的回滚判断逻辑：

判断条件：
1. 稳定版是否有有效备份？（backup_verified=1）
2. 是否有可用 Seal Artifact？（package_present=1）
3. 是否有最近恢复演练记录？（30 天内成功 drill）
4. 是否存在阻塞级 Incident？（incident.status=blocked）
5. 是否存在活跃 Gate 阻塞？（gate.blocking_status IS NOT NULL）

输出状态：
- **ready**: 满足全部条件，可随时回滚
- **caution**: 备份有效但演练较旧或有轻微告警
- **blocked**: 存在阻塞问题（无备份、有活跃 incident、Gate 阻塞）

目标：
把"能不能回滚"从经验判断推进为系统可见判断。

### D. Release Validation 聚合
对发布相关验证结果做聚合展示：

聚合字段：
- gate_passed_count / gate_total_count
- validation_passed_count / validation_total_count
- incident_count (关联 incident 数)
- recovery_drill_count (最近恢复演练次数)
- health_status: ok / warning / error

目标：
发布不再只是一个节点，而是带验证结果的对象。

### E. 发布异常联动闭环
打通以下链路：
- Release → Backup Artifact（可跳转查看）
- Release → Seal Manifest（可跳转查看）
- Release → Recovery Drill（可查看演练记录）
- Release → Incident（可查看关联异常）
- Release → Audit（可查看审计日志）
- Candidate → Stable promotion trace（可查看晋升轨迹）

目标：
从发布对象出发，能一路追到相关封板、备份、恢复和异常。
