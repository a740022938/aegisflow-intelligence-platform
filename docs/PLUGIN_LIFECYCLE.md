# 插件生命周期管理文档 (初稿)

## 1. 插件生命周期阶段

### 1.1 注册阶段 (Registration)
- 插件在启动时扫描 `plugins/builtin` 目录
- 读取插件的 `manifest.json` 或代码中的元信息
- 验证插件接口是否符合规范
- 将插件信息存入 `plugin_registry` 表

### 1.2 初始化阶段 (Initialization)
- 只在启动时执行一次 (startup-only init)
- 执行插件的 `init()` 方法
- 加载模型、资源、配置
- 记录初始化结果到 `plugin_init_runs` 表

### 1.3 运行阶段 (Runtime)
- 插件处理请求
- 实现熔断机制 (circuit breaker)
- 超时控制
- 资源配额限制

### 1.4 卸载/禁用阶段 (Unload/Disable)
- 插件可以被禁用
- 禁用后不再接收请求
- 资源被释放

## 2. 插件状态

| 状态 | 说明 |
|------|------|
| registered | 已注册，未初始化 |
| initializing | 初始化中 |
| ready | 就绪，可处理请求 |
| error | 初始化或运行出错 |
| disabled | 已禁用 |

## 3. 插件写回规范

### 3.1 允许的写回操作
- 通过主程序提供的标准 API 写入结果
- 写入自己的插件私有表
- 更新插件状态

### 3.2 禁止的操作
- 直接修改主程序的核心表 (tasks, datasets, models 等)
- 绕过审计日志
- 写入其他插件的私有数据

### 3.3 审计要求
- 所有插件操作必须记录到 `plugin_events` 表
- 记录操作类型、操作者、时间戳
- 保留来源链以便追溯

## 4. 插件开发约束

### 4.1 超时控制
- 默认超时: 60 秒
- 可在 manifest 中配置自定义超时

### 4.2 资源配额
- 内存限制: 2GB (可配置)
- 并发请求数: 10 (可配置)

### 4.3 熔断机制
- 连续失败 5 次后自动熔断
- 熔断后 30 秒尝试恢复
- 熔断信息可在 `/api/plugins` 查看
