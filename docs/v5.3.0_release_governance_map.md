## 2. 对象关系详解

### A. Release 表结构（既有）

```
releases:
  id, artifact_id, model_id, release_name, release_version,
  status (draft/candidate/sealed/archived),
  sealed_by, sealed_at, release_notes,
  release_manifest_json, source_evaluation_id, source_experiment_id,
  source_dataset_id, metrics_snapshot_json,
  approval_id, approval_status,
  package_present (0/1), backup_verified (0/1),
  created_at, updated_at
```

**关键字段解读**：
- `status`: draft → candidate → sealed → archived
- `package_present`: 是否有封板产物（Seal Artifact）
- `backup_verified`: 备份是否已验证
- `approval_id`: 关联审批记录
- `release_manifest_json`: 封板清单

### B. Backup 关联

```
release --1:1--> seal_artifact (通过 package_present 标识)
release --1:N--> recovery_logs (source_release = release.id)
release --1:N--> audit_logs (category = 'release', target = artifact_id)
```

### C. Gate 关联

```
gate_checks:
  entity_type = 'release' / 'artifact'
  entity_id = release.id / artifact.id
  status = 'passed' / 'blocked'
  blocking_status IS NOT NULL 表示当前阻塞
```

### D. Incident 关联

```
incident:
  related_release_id = release.id
  incident_type = 'release_issue'
  status = 'active' / 'resolved'
```

## 4. Rollback Readiness 判断规则

### 判断条件

| 检查项 | 条件 | 通过条件 |
|--------|------|----------|
| backup_verified | release.backup_verified = 1 | true |
| package_present | release.package_present = 1 | true |
| recent_drill | 30 天内有成功 recovery_logs | drill.status = 'success' |
| no_blocking_incident | 无 status = 'active' 的 incident | incident_count = 0 |
| no_active_gate_block | 无 blocking_status IS NOT NULL 的 gate_check | blocked_count = 0 |

### 状态输出

- **ready**: 全部 5 项通过
- **caution**: backup_verified + package_present 通过，但其他有告警
- **blocked**: 关键条件不满足（无备份、有活跃 incident）

## 5. Candidate → Stable 晋升判断

### 准备条件

| 检查项 | 条件 |
|--------|------|
| all_gates_passed | 所有 gate_checks.status = 'passed' |
| no_active_incident | incident.status != 'active' |
| approval_completed | approval.status = 'approved' |
| validation_passed | validation_summary.passed = validation_summary.total |

### 晋升状态

- **promotion_ready**: 全部条件满足
- **promotion_blocked**: 存在阻塞项
- **promotion_pending**: 等待审批或验证

## 8. SQL 查询示例

### A. 获取 Stable Release

```sql
SELECT * FROM releases 
WHERE status = 'sealed' 
ORDER BY sealed_at DESC 
LIMIT 1;
```

### B. 获取 Candidate Release

```sql
SELECT * FROM releases 
WHERE status = 'candidate' 
ORDER BY created_at DESC 
LIMIT 1;
```

### C. 检查 Rollback Readiness

```sql
-- 检查备份
SELECT backup_verified, package_present FROM releases WHERE id = ?;

-- 检查最近演练
SELECT * FROM recovery_logs 
WHERE source_release = ? 
  AND status = 'success' 
  AND performed_at > datetime('now', '-30 days')
ORDER BY performed_at DESC LIMIT 1;

-- 检查活跃 incident
SELECT COUNT(*) FROM incidents 
WHERE related_release_id = ? AND status = 'active';

-- 检查 Gate 阻塞
SELECT COUNT(*) FROM gate_checks 
WHERE entity_id = ? AND blocking_status IS NOT NULL;
```

### D. 发布验证聚合

```sql
-- Gate 汇总
SELECT gate_name, status FROM gate_checks 
WHERE entity_id = ? AND entity_type = 'release';

-- 关联 audit
SELECT * FROM audit_logs 
WHERE category = 'release' AND target = ?
ORDER BY created_at DESC LIMIT 10;

-- 关联 recovery
SELECT * FROM recovery_logs 
WHERE source_release = ?
ORDER BY performed_at DESC LIMIT 5;
```
