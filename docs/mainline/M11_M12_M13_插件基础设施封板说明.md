# M11-M13 插件基础设施封板说明

## 目标
- M11：收紧 discovery 入口（Manifest/Schema）。
- M12：增加 activation 前 package health 预检。
- M13：将启动、熔断、cache、registry、events、enable/disable、isolation 固化为主线基线（不进入 execute）。

## 架构基线（当前）
1. Discovery 层
- 扫描插件目录并读取 `manifest.json`。
- 执行强 schema 校验，不合规直接拒绝。

2. Registry 层
- 读取 persisted enabled 覆盖。
- 记录插件基础信息与初始化状态（`plugin_registry`）。

3. Activation Precheck 层（M12）
- 对 enabled 插件执行启动前预检：entry、路径边界、文件存在性、core 版本兼容。
- 预检失败写 `precheck_failed` + error_reason。

4. Activation/Catalog 层
- 仅预检通过插件进入激活集合。
- 仅激活集合参与 catalog cache。
- `/api/vision/catalog` 始终从 cache + builtin fallback 返回。

## 关键约束
1. 普通请求链路不触发插件初始化。
2. 禁止每请求 import/new PluginManager。
3. execute 继续关闭（503）。

## 数据面与审计
1. registry：`plugin_registry`
- 复用 `init_status/error_reason/active` 追踪预检结果。
2. 启动摘要：`plugin_init_runs`
- 记录 discovered/success/failed 计数。
3. 事件审计：`plugin_events` + `audit_logs`
- 记录 enable/disable 与启动期审计。

## 验证摘要
1. 正常启动：health/plugins/status/vision/catalog 正常。
2. 临时坏插件注入：
- discovered 增加
- failed 增加
- registry 记录 `precheck_failed`
- catalog 不包含坏插件
3. execute 仍 503。

## 边界声明
- 本阶段不实现 execute。
- 本阶段不改 workflow/index.ts 与视觉执行层（YOLO/SAM/Tracker/Rule Engine）。
