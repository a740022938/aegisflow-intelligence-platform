## 六、M5：TS5011 Build修复

**问题描述：**
pnpm build -> tsc --outDir dist -> TS5011

**根因（TypeScript 6.0.2）：**
- index.ts:2783：await import(../../../packages/plugin-runtime/src/PluginManager.js)
- tsc编译时将plugin-runtime下的文件也纳入输入
- rootDir推断为../..（monorepo根），与include: src/**/*.ts冲突

**修复方案：**
apps/local-api/tsconfig.json: 新增 noEmit: true + outDir: "dist"

**89个历史type errors：**
- 一直存在，被TS5011遮蔽
- 独立技术债（Zod API变更、any类型滥用等）
- 不影响tsx运行，不阻断开发

## 八、关键教训

1. PowerShell变量与ANSI转义码冲突：inline $var 在带颜色的exec输出中会被PTY转义序列干扰，脚本文件是解
2. DB路径误判：M1扫描时扫到data/下的空DB，以为生产DB也是空的，实际生产DB在packages/db/
3. Build被TS5011遮蔽：tsc遇到第一个错误就停止，89个type errors从来看不到
4. pre-restore备份泛滥：M系列备份每轮保存9个版本，单次操作就产生9个
5. GPU训练资源：session gentle-falcon中CUDA PyTorch安装成功（此前CPU-only版本），GPU可用
