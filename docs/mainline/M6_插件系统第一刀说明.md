# M6 插件系统第一刀说明

## 概述

**M6** 是 AGI Model Factory 插件系统升级的第一刀，目标是将 vision-bus 的硬编码 catalog 发现层升级为通过 PluginManager 动态发现的正式入口。

## 核心变更

### 1. 插件系统默认启用

- **修改前**: 插件系统通过环境变量 `PLUGIN_SYSTEM_ENABLED=true` 控制，默认关闭
- **修改后**: 插件系统默认启用，不再依赖环境变量

### 2. vision-bus catalog 插件化

- **修改前**: `getVisionCatalog()` 返回硬编码的 `BUILTIN_PIPELINES`
- **修改后**: `getVisionCatalog()` 异步合并内置管线 + 插件发现的 vision 能力

### 3. 新增 vision capability

plugin-sdk 新增 `'vision'` 能力类型，用于标识视觉处理插件：

```typescript
export type Capability = 'report' | 'read' | 'compute' | 'notify' | 'transform' | 'export' | 'vision';
```

## 插件标签约定

vision 插件通过 `manifest.tags` 声明管线属性：

| 标签 | 用途 |
|------|------|
| `step:xxx` | 声明 workflow_steps |
| `artifact:xxx` | 声明 output_artifact_types |
| `table:xxx` | 声明 intermediate_tables |
| `frozen` | 声明 status=frozen |
| `planned` | 声明 status=planned |

### 示例插件 manifest

```json
{
  "plugin_id": "custom-detector",
  "name": "自定义检测器",
  "version": "1.0.0",
  "capabilities": ["vision"],
  "risk_level": "MEDIUM",
  "tags": [
    "step:custom_detect",
    "artifact:custom_detection",
    "table:custom_detections"
  ]
}
```

## API 变更

### GET /api/vision/catalog

**变更**: 返回值变为 Promise，支持异步插件发现

**响应格式**:

```json
{
  "ok": true,
  "version": "1.1.0-m6",
  "pipelines": [...],
  "artifact_type_registry": [...],
  "total_pipelines": 9,
  "active_pipelines": 3,
  "_meta": {
    "builtin_count": 9,
    "plugin_count": 0,
    "plugin_system_active": true
  }
}
```
