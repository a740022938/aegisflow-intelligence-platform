# M15 官方视觉能力边界表

## 能力边界表

| 边界项 | 当前结论 | 约束说明 |
|---|---|---|
| 管理层桥接范围 | 6 项官方视觉能力全部处于管理层桥接视图 | 仅 discovery/registry/status/catalog 可见，不等于 execute 可用 |
| 不可执行能力 | vision-yolo / vision-sam / vision-mahjong-classifier / vision-tracker / vision-rule-engine / vision-fusion | execute 接口统一关闭（503），当前不得打开 |
| 不可直接接入 workflow | 全部 6 项 | 未允许修改 `workflow/index.ts`，禁止把桥接能力直接挂到 workflow 执行链 |
| planned 能力边界 | vision-tracker / vision-rule-engine / vision-fusion | 仅占位，不可激活为执行能力，不可宣称已交付执行实现 |
| builtin 能力边界 | vision-yolo / vision-sam / vision-mahjong-classifier | 仅表示官方内置能力已纳入管理视图，不代表已开放插件 execute |

## 进入执行层的统一前置门禁
1. 单能力执行隔离设计完成（资源、超时、错误分级、熔断）。
2. 单能力回归用例通过（成功路径 + 失败路径 + 降级路径）。
3. 主程序稳定性验证通过（health/plugins/status/vision/catalog 全部正常）。
4. 审计可追踪（事件、错误原因、状态回写完整）。
5. 经过单刀验收后，才允许讨论 execute 开关。

## 禁止清单

### 1) 当前绝对不能乱动的文件/模块
1. `apps/local-api/src/workflow/index.ts`
2. `apps/local-api/src/task-execute/*`
3. YOLO/SAM/Classifier 主程序现有执行实现路径

### 2) 当前不能提前打开的能力
1. `POST /api/plugins/:id/execute`
2. tracker/rule_engine/fusion 的执行链路
3. 把 planned_official 误当成可执行插件

### 3) 当前不能误认为已完成的内容
1. “桥接完成”不等于“执行完成”。
2. “catalog 可见”不等于“workflow 可调用”。
3. “builtin_official”不等于“已开放 execute”。
