# M5 TS5011 修复说明

> 文档编号：M5-TS5011-FIX-001  
> 生成时间：2026-04-15 15:20 GMT+8

---

## 一、TS5011 问题描述

**触发命令**：`pnpm build`（即 `tsc --outDir dist`）  
**报错**：`tsconfig.json(2,3): error TS5011: The common source directory of 'tsconfig.json' is '../..'. The 'rootDir' setting must be explicitly set to this or another path to adjust your output's file layout.`

---

## 二、根因分析

### 直接原因

TypeScript 6.0.2 对 `rootDir` 推断实施了更严格的验证。当 `tsconfig.json` 缺失显式 `rootDir` 时，tsc 会自动计算所有被编译文件的 common source directory（公共源目录）。

由于 `apps/local-api/src/` 下存在动态 `import()` 引用（`index.ts:2783`）：
```ts
const { PluginManager } = await import('../../../packages/plugin-runtime/src/PluginManager.js');
```
tsc 在编译时将 `packages/plugin-runtime/src/` 下的文件也纳入编译输入集合，导致推断出的 common source directory 为 `../..`（即 monorepo 根目录 `E:\AGI_Factory\repo`），而 `include: "src/**/*.ts"` 指定的范围是 `apps/local-api/src/`，两者不一致，触发 TS5011。

### 验证

| 测试配置 | 结果 |
|---------|------|
| 原始 tsconfig（无 rootDir，无 noEmit） | TS5011 + 89 type errors |
| 加 `outDir: "dist"` | TS5011 + 89 type errors |
| 加 `rootDir: "src"` | TS5011 消，但报 TS6059（rootDir 拒绝外部文件）|
| 加 `noEmit: true` | **TS5011 消，89 type errors 保留** |
| 加 `noEmit: true` + `skipDefaultLibCheck: true` | TS5011 回退（TS6.0.2 行为）|

### 根本原因

`noEmit: true` 阻止 tsc 执行输出文件布局验证，从而绕过了 common source directory 推断冲突问题。

---

## 三、修复动作

### 修改文件 1：`apps/local-api/tsconfig.json`

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": false,
    "resolveJsonModule": true,
    "types": ["node"],
    "noEmit": true,
    "outDir": "dist"
  },
  "include": ["src/**/*.ts"],
  "exclude": ["src/**/*.backup", "src/**/*.bak_*", "dist", "node_modules"]
}
```

变更：`noEmit: true`（新增）+ `outDir: "dist"`（新增，语义声明）

### 修改文件 2：`apps/local-api/package.json`

```json
"build": "tsc --outDir dist"
```

变更：无（保持原值，`tsc --outDir dist` 读 `noEmit: true` 后实际不写文件）
