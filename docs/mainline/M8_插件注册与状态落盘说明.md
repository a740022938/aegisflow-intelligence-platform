# M8 插件注册与状态落盘说明

## 背景
M7 已完成：启动期初始化 + 熔断 + catalog cache。
M8 在此基础上补齐“管理层”数据底座：插件注册落盘、初始化摘要落盘、只读状态查询。

## 设计原则
1. 不改请求期初始化策略（仍禁止每请求初始化）
2. 不改视觉实现层与 workflow 执行器
3. 先保证主程序稳定，再增强可观测性

## 数据模型
### 新增 `plugin_registry`
承载插件当前注册快照：
- 标识与基础信息：`plugin_id`, `plugin_name`, `version`, `capability`, `source`
- 状态：`enabled`, `active`, `init_status`, `error_reason`
- 时间：`discovered_at`, `initialized_at`, `created_at`, `updated_at`

### 新增 `plugin_init_runs`
承载启动期初始化摘要：
- `id`, `init_status`
- `plugin_system_enabled`, `plugin_system_active`
- `discovered_count`, `success_count`, `failed_count`
- `error_summary`
- `started_at`, `finished_at`, `created_at`

### 复用 `audit_logs`
新增插件启动审计动作：`plugin_startup_discovery`，记录数量与错误摘要。

## 启动期流程（M8）
1. 执行 M7 启动期初始化（仅一次）
2. 成功/失败结果写入运行时状态
3. 将本次快照落盘：`plugin_registry` + `plugin_init_runs`
4. 写审计：`audit_logs`
5. `vision/catalog` 仍只读 cache，不触发初始化

## 接口（只读）
1. `GET /api/plugins`
- 返回持久化插件列表 + 运行时状态 + 最新初始化摘要

2. `GET /api/plugins/:id`
- 返回单插件持久化状态

3. `GET /api/plugins/status`
- 返回当前运行时状态（enabled/active/init_failed/error_reason/plugin_count 等）
- 同时返回 `init_summary`

## 失败追踪能力
- 当插件目录配置错误或启动初始化异常时：
  - `plugin_init_runs.error_summary` 保留错误原因
  - `plugin_registry.error_reason` 更新为当前错误
  - `audit_logs.detail_json` 同步记录错误摘要

## 兼容性
- 未改变 `/api/vision/catalog` 的 cache 返回逻辑。
- 未引入请求期初始化回退。
- 未触碰 YOLO 与 workflow 实现层。
