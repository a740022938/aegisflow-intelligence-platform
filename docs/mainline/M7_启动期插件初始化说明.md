# M7 启动期插件初始化说明

## 设计目标
M7 仅重接插件发现层，不重开请求期初始化：
- 初始化只在服务启动时执行一次
- 普通请求只读缓存
- 初始化失败时熔断并回退 builtin catalog

## 架构变化

## 1) 启动期单次初始化
`apps/local-api/src/index.ts` 在 `start()` 中调用：
- `visionBus.initializeVisionPluginSystemAtStartup({ enabled, pluginDir, logLevel })`

行为：
- 成功：`plugin_system_active=true`，缓存可用。
- 失败：`init_failed=true`，记录 `error_reason`，服务继续启动。

## 2) 熔断状态机（vision-bus 内）
`apps/local-api/src/vision-bus/index.ts` 增加运行时状态：
- `plugin_system_enabled`
- `plugin_system_active`
- `init_attempted`
- `init_failed`
- `error_reason`
- `plugin_count`
- `init_completed_at`

熔断规则：
- `init_attempted=true` 后，后续不再执行初始化逻辑。
- 初始化失败后，持续返回失败状态，不重试。

## 3) catalog cache
`vision-bus` 增加：
- `visionCatalogCache`
- `rebuildVisionCatalogCache()`
- `buildVisionCatalog(pluginPipelines)`

`/api/vision/catalog` 只返回 `visionCatalogCache`，不触发 import/new PluginManager。

## 4) 只读状态接口
`/api/plugins/status` 输出：
- `plugin_system_enabled`
- `plugin_system_active`
- `init_failed`
- `error_reason`
- `plugin_count`
- `init_attempted`
- `init_completed_at`

## 5) 读写边界
M7 期间 `/api/plugins/:id/enable|disable|execute` 固定返回 503（read-only mode），防止误把插件运行时控制逻辑重新接入请求热路径。

## 非目标（保持不变）
- 不改 `workflow/index.ts`
- 不改 YOLO 实现层
- 不扩展 tracker/rule_engine/fusion
- 不触发麻将训练链路
