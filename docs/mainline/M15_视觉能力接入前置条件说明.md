# M15 视觉能力接入前置条件说明

## 范围
本说明用于后续 tracker / rule_engine / fusion / 数据飞轮相关能力的接入前门禁，不改变当前 execute 关闭语义。

## 接入前置条件（必须同时满足）
1. 身份与边界明确
- 能力已在官方清单登记（plugin_id/source_type/status/risk_level/managed_by）。
- current_role 明确为“管理层桥接”或“执行候选”。

2. 运行安全门禁
- 启动期初始化与请求期隔离保持不变。
- 失败熔断、错误原因、状态回写可用。
- 不得引入每请求初始化。

3. 可观测与审计
- 状态可通过 `/api/plugins` 与 `/api/plugins/status` 观察。
- 变更事件与审计记录可追踪。
- `init_status/error_reason` 语义保持一致。

4. 稳定性回归
- `GET /api/health` 正常。
- `GET /api/plugins` 正常。
- `GET /api/plugins/status` 正常。
- `GET /api/vision/catalog` 正常。

5. 变更边界遵守
- 不改 `workflow/index.ts`。
- 不迁移 YOLO 出主程序。
- 不扩展 tracker/rule_engine/fusion 执行实现（在未获准前）。

## 分阶段建议（进入 M16 参考）
1. 先做制度化增强：状态语义、审计维度、文档一致性。
2. 再做单能力候选评估：从 `vision-sam` 开始比从 planned 能力开始风险更低。
3. 最后才讨论 execute 灰度开关（必须单刀验收）。
