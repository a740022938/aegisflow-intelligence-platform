# M10 插件加载隔离说明

## 背景
M9 已完成控制面 enable/disable 与事件审计。M10 目标是加强加载隔离，明确 discovery/registry/activation 分层，避免 disabled 插件进入实际激活路径。

## 分层设计

## 1) Discovery（发现）
- 扫描插件目录子目录
- 解析 `manifest.json`
- 形成 discovered plugin 列表
- 此阶段不执行插件逻辑

## 2) Registry（注册）
- 读取 `plugin_registry.enabled` 覆盖
- 形成最终插件状态（enabled/disabled）
- 启动结果继续落盘到：
  - `plugin_registry`
  - `plugin_init_runs`
  - `audit_logs(plugin_startup_discovery)`

## 3) Activation（激活）
- 仅 enabled 插件进入 activation 集合
- 仅 enabled 插件参与 catalog/cache 构建
- disabled 插件被隔离在 activation 之外

## 与 M9 控制面的关系
- enable/disable 接口仍仅修改 registry 状态，不执行插件逻辑
- 状态变更继续写入：
  - `plugin_events`
  - `audit_logs(plugin_enabled/plugin_disabled)`

## API 兼容性
保持可用：
- `/api/plugins`
- `/api/plugins/:id`
- `/api/plugins/status`
- `/api/vision/catalog`
- `/api/health`

## 执行层边界
- `/api/plugins/:id/execute` 继续 503 拦截
- 本轮未引入 execute、升级、回滚能力
