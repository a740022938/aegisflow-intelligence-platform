# M14 官方视觉模块桥接说明

## 背景
M11/M12/M13 已封板，插件系统当前定位为管理层能力（discovery/registry/status/control），执行层继续关闭。

## 本轮设计
在不迁移主程序视觉实现（YOLO/SAM/Classifier）的前提下，增加“官方桥接插件记录”，让内置能力进入插件管理视角。

## 桥接分层
1. builtin_official（可纳入激活）
- vision-yolo
- vision-sam
- vision-mahjong-classifier

2. planned_official（仅占位）
- vision-tracker
- vision-rule-engine
- vision-fusion

## 运行规则
1. 启动期 discovery 完成后注入官方桥接清单。
2. enabled 且 precheck 通过的项才进入 activation/catalog plugin pipeline。
3. planned_official 默认 disabled，不进入 activation。
4. execute 仍由 `/api/plugins/:id/execute` 统一 503 拦截。

## 对外可见性
1. `/api/plugins`
- 返回 persisted plugin 列表
- 同时返回 `official_bridges`（标准桥接字段 + 运行态状态）

2. `/api/plugins/status`
- 返回 runtime/init_summary
- 同时返回 `official_bridges`

3. `/api/vision/catalog`
- `_meta.official_bridges` 返回桥接映射摘要

## 边界声明
1. YOLO/SAM/Classifier 实现仍在主程序内。
2. 本轮不启用 execute。
3. 本轮不改 workflow/index.ts。
4. 本轮不扩展 tracker/rule_engine/fusion 执行实现。
