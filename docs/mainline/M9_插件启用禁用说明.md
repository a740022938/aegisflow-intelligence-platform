# M9 插件启用禁用说明

## 背景
M8 已完成插件注册落盘与状态查询。M9 在不打开执行层的前提下，补齐控制面第一刀：启用/禁用 + 事件审计。

## 数据模型
### 新增表：`plugin_events`
记录插件状态变更事件：
- `plugin_id`, `plugin_name`
- `action`（enable / disable）
- `before_status`, `after_status`
- `reason`
- `created_at`

### 复用表
- `plugin_registry`：维护 `enabled/active` 当前状态
- `audit_logs`：写入控制面审计动作

## 写接口
1. `POST /api/plugins/:id/enable`
2. `POST /api/plugins/:id/disable`

行为约束：
- 仅改 registry 状态（不执行插件逻辑）
- 写 plugin_events
- 写 audit_logs
- 返回最新插件状态 + 本次事件

## 启动期筛选
- 启动期初始化会读取 registry 的 `enabled` 覆盖。
- 仅 enabled 插件进入“启用集合”。
- disabled 插件不参与 vision pipeline cache。

## 只读接口（延续 + 补充）
- `GET /api/plugins`
- `GET /api/plugins/:id`
- `GET /api/plugins/status`
- `GET /api/plugins/events`
- `GET /api/plugins/:id/events`

## 执行层边界
- `POST /api/plugins/:id/execute` 仍固定拦截（503）。
- 本轮未实现 upgrade/rollback，未触碰 workflow 与视觉实现层。
