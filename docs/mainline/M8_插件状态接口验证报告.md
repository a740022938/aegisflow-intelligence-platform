# M8 插件状态接口验证报告

## 验证范围
- `GET /api/plugins`
- `GET /api/plugins/:id`
- `GET /api/plugins/status`
- 主程序健康与 catalog 稳定性回归

## 环境
- Node: `v22.22.2`
- 启动目录: `apps/local-api`
- 数据库: `packages/db/agi_factory.db`

## 接口验证结果
1. `GET /api/plugins`
- 返回 `ok=true`
- 返回持久化插件列表 `total=1`
- 返回 `runtime` 与 `init_summary`

2. `GET /api/plugins/builtin-demo-plugin`
- 返回插件记录：
  - `plugin_name=Demo Plugin`
  - `version=1.0.0`
  - `capability=report,read`
  - `source=builtin`
  - `enabled=true`
  - `active=true`
  - `init_status=success`

3. `GET /api/plugins/status`
- 返回：
  - `plugin_system_enabled=true`
  - `plugin_system_active=true`
  - `init_failed=false`
  - `error_reason=null`
  - `discovered_count=1`
  - `init_success_count=1`
  - `init_failed_count=0`
  - `init_summary`（含 started/finished、success/failed 数量）

## 数据库一致性验证
1. `plugin_registry`
- 与接口返回一致（demo 插件 1 条）

2. `plugin_init_runs`
- 最新一条为 success
- 历史可见 failed 记录（用于故障追踪）

3. `audit_logs`
- 可见 `plugin_startup_discovery`，含 discovered/success/failed/error 摘要

## 回归验证
1. `GET /api/health` -> `ok=true`
2. `GET /api/vision/catalog` -> `ok=true`，且未破坏现有逻辑
3. 连续访问 `/api/plugins` 与 `/api/vision/catalog` 后，`init_run_id` 不变（未发生请求期重复初始化）

## 结论
M8 接口层满足验收标准：
- 注册信息可查
- 初始化状态可查
- 启动错误可追踪
- 主程序保持稳定
