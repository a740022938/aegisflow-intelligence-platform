# M9 插件控制面验证报告

## 验证范围
1. enable/disable 写接口
2. plugin_events 事件落盘
3. audit_logs 审计落盘
4. 启动期按 enabled 筛选
5. 主程序与 catalog 稳定性

## 结果摘要

## 1) enable/disable 可用
- `POST /api/plugins/builtin-demo-plugin/disable` -> `ok=true`
- `POST /api/plugins/builtin-demo-plugin/enable` -> `ok=true`
- 均返回最新 `plugin` 状态与 `event`

## 2) 事件与审计落盘
- `plugin_events` 中出现连续事件：
  - disable（enabled -> disabled）
  - enable（disabled -> enabled）
- `audit_logs` 中出现：
  - `plugin_disabled`
  - `plugin_enabled`

## 3) 启动期筛选生效
- 在插件 disabled 状态下重启：
  - `discovered_count=1`
  - `init_success_count=0`
- 说明：发现到了插件，但未纳入 enabled 加载集合。

## 4) 稳定性回归
- `/api/health` 正常
- `/api/vision/catalog` 正常
- `/api/plugins/status` 正常
- `execute` 仍被 503 拦截

## 5) 结论
M9 第一刀达成：
- 控制面启用/禁用已具备
- 状态变更事件可审计、可追踪
- 启动期 enabled 筛选生效
- 主程序稳定
