## 1. 目标

解决 B4 中发现的"100% accept 但置信度极低"问题，建立可靠的分类器过滤能力。

**核心问题**:
- B4 结果: 6/6 accepted, avg confidence = 0.053 (过低)
- 需要: 明确的 accept/reject 规则 + 可验证的负样本拒绝能力

---

## 2. 阈值校准方案

### 2.1 当前发现

**正样本** (真实麻将图片):
- Min conf: 0.0415
- Max conf: 0.0533
- Avg conf: 0.0457

**负样本** (合成: 纯色/噪声/渐变/UI按钮):
- Min conf: 0.0379
- Max conf: 0.0469
- Avg conf: 0.0415

**问题**: 正负样本置信度分布重叠 (0.0415-0.0469 重叠区间)

### 2.2 根本原因

| 因素 | 影响 |
|------|------|
| 正样本为完整场景图 | 非单独牌面，置信度偏低 |
| 负样本为合成图片 | 与真实 UI/背景差异大 |
| 模型训练数据 | Mahjong Soul 游戏截图，与当前测试数据 domain gap |

### 2.3 建议阈值

| 阈值 | 正样本接受率 | 负样本拒绝率 | 适用场景 |
|------|-------------|-------------|---------|
| 0.03 | 100% | 0% | 宽松 (高召回) |
| **0.04** | **100%** | **42%** | **平衡 (推荐)** |
| 0.05 | 20% | 100% | 严格 (高精度) |
| 0.044 | 50% | 50% | 不建议 (重叠区) |

**推荐**: threshold = 0.04
- 保证正样本高接受率 (100%)
- 对明显负样本 (纯色) 有拒绝能力 (42%)

### 4.1 旧链路 (YOLO 单模型)

|------|-----|------|

## 5. 主链修正

### 5.1 classifier_results 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| predicted_label | TEXT | 预测类别 (如 5n) |
| confidence | REAL | 预测置信度 |
| is_accepted | INTEGER | 0/1 |
| rejection_reason | TEXT | low_confidence / null |

### 5.2 Lineage 可见性

**Accept 流程**:
```
yolo_annotation → classifier_result (is_accepted=1) → review_pack → dataset_version
```

**Reject 流程**:
```
yolo_annotation → classifier_result (is_accepted=0, rejection_reason=low_confidence) → negative_pool (可选)
```

**查询示例**:
```sql
-- 查看 reject 分流
SELECT * FROM classifier_results WHERE is_accepted = 0;

-- 查看 accept 进入下游的
SELECT * FROM classifier_results WHERE is_accepted = 1;
```

## 6. 限制与建议

### 6.1 当前限制

1. **正样本置信度偏低**: 测试用完整场景图，非单独牌面 crop
2. **负样本为合成**: 与真实 UI/背景有差异
3. **阈值敏感**: 0.04→0.05 导致正样本接受率从 100% 降至 20%

### 6.2 改进建议

1. **使用真实 YOLO crops**: 正样本用 YOLO 检测后的裁剪图
2. **收集真实负样本**: 从游戏截图中提取 UI/背景区域
3. **模型微调**: 在当前 domain 数据上微调 mahjong_vision
4. **多阈值策略**: 不同场景使用不同阈值
